---
title: "Image Support"
---

Flock supports image analysis in SQL through the same `context_columns` abstraction used for text and audio. Use vision-capable models with `type: 'image'` to describe, filter, and aggregate over image data alongside tabular columns.

For audio workflows, see [Audio Transcription](/audio-support). For batching and cost tuning, see [Performance and cost](/performance).

## Quick start

```sql
SELECT
    product_name,
    llm_complete(
        {'model_name': 'gpt-4o'},
        {
            'prompt': 'Describe this product image briefly.',
            'context_columns': [
                {'data': image_url, 'type': 'image'}
            ]
        }
    ) AS description
FROM VALUES
    ('Wireless Headphones', 'https://example.com/headphones.jpg')
AS t(product_name, image_url);
```

## Supported formats and providers

| Format | Extensions |
|--------|------------|
| JPEG | `.jpg`, `.jpeg` |
| PNG | `.png` |
| GIF | `.gif` |
| WebP | `.webp` |
| BMP | `.bmp` |

| Provider | Supported inputs |
|----------|------------------|
| **OpenAI** vision | HTTP/HTTPS URLs, base64 strings |
| **Ollama** vision | Base64 strings |
| **Anthropic / Claude** | URLs (auto-downloaded), file paths, base64 |

## Context columns API

```sql
'context_columns': [
  {'data': column_name, 'type': 'image'},
  {'data': column_name, 'type': 'image', 'detail': 'low'}  -- OpenAI only
]
```

| Property | Description |
|----------|-------------|
| **`data`** _(required)_ | URL, file path, or base64 image data |
| **`name`** _(optional)_ | Alias referenced in prompts |
| **`type`** | Set to `'image'` |
| **`detail`** _(OpenAI only)_ | `'low'` (default), `'medium'`, or `'high'` — controls token usage vs accuracy |

<Accordion title="llm_complete — descriptions and marketing copy">

```sql
-- Image description
SELECT
    product_name,
    llm_complete(
        {'model_name': 'gpt-4o'},
        {
            'prompt': 'Describe this product image in detail.',
            'context_columns': [{'data': image_url, 'type': 'image'}]
        }
    ) AS image_description
FROM product_images;

-- Combine image with text context
SELECT
    llm_complete(
        {'model_name': 'gpt-4o'},
        {
            'prompt': 'Write marketing copy for {product} in category {category}.',
            'context_columns': [
                {'data': product_name, 'name': 'product'},
                {'data': category, 'name': 'category'},
                {'data': image_url, 'type': 'image'}
            ]
        }
    ) AS marketing_copy
FROM product_catalog;
```

<Accordion title="llm_filter — visual criteria">

```sql
SELECT *
FROM photo_uploads
WHERE llm_filter(
    {'model_name': 'gpt-4o'},
    {
        'prompt': 'Is this an outdoor landscape photograph?',
        'context_columns': [{'data': photo_url, 'type': 'image'}]
    }
);

-- Filter with text + image context
SELECT product_id, product_name, price
FROM products
WHERE llm_filter(
    {'model_name': 'gpt-4o'},
    {
        'prompt': 'Is this a professional product photo with good lighting?',
        'context_columns': [
            {'data': image_url, 'type': 'image'},
            {'data': product_name}
        ]
    }
)
AND price > 100;
```

</Accordion>

<Accordion title="Embeddings via image descriptions">

`llm_embedding` supports text only. Generate descriptions first, then embed:

```sql
WITH image_descriptions AS (
    SELECT
        image_id,
        llm_complete(
            {'model_name': 'gpt-4o'},
            {
                'prompt': 'Describe this image in detail.',
                'context_columns': [{'data': image_url, 'type': 'image'}]
            }
        ) AS generated_description
    FROM image_catalog
)
SELECT
    image_id,
    generated_description,
    llm_embedding(
        {'model_name': 'text-embedding-3-small'},
        {'context_columns': [{'data': generated_description}]}
    ) AS description_embedding
FROM image_descriptions;
```

</Accordion>

<Accordion title="Aggregate functions — llm_first, llm_reduce">

```sql
-- Best product image per category
SELECT
    category,
    llm_first(
        {'model_name': 'gpt-4o'},
        {
            'prompt': 'Which product has the most appealing product image?',
            'context_columns': [
                {'data': product_name},
                {'data': image_url, 'type': 'image'}
            ]
        }
    ) AS best_product
FROM products
GROUP BY category;

-- Summarize themes across an image album
SELECT
    album_id,
    llm_reduce(
        {'model_name': 'gpt-4o'},
        {
            'prompt': 'What visual themes appear across these images?',
            'context_columns': [
                {'data': image_url, 'type': 'image'},
                {'data': caption}
            ]
        }
    ) AS album_summary
FROM album_images
GROUP BY album_id;
```

</Accordion>

## Performance

Use `max_batch_size` when processing many images, and prefer `'detail': 'low'` on OpenAI unless you need fine-grained analysis:

```sql
SELECT
    image_id,
    llm_complete(
        {'model_name': 'gpt-4o', 'max_batch_size': 5},
        {
            'prompt': 'Describe this image briefly.',
            'context_columns': [{'data': image_url, 'type': 'image'}]
        }
    ) AS description
FROM image_catalog
LIMIT 100;
```

See [Performance and cost](/performance) for `is_async`, `rate_limit`, and metrics workflows.

## Function support

| Function | Image support | Description |
|----------|---------------|-------------|
| `llm_complete` | Yes | Generate text from image content |
| `llm_filter` | Yes | Filter rows by visual criteria |
| `llm_embedding` | Text only | Embed text derived from images (two-step workflow) |
| `llm_reduce` | Yes | Summarize image collections |
| `llm_rerank` | Yes | Rank by visual relevance |
| `llm_first` | Yes | Select top match by visual criteria |
| `llm_last` | Yes | Select bottom match by visual criteria |
