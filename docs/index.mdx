# Beyond Quacking: Deep Integration of Language Models and RAG into DuckDB

Anas Dorbani · Sunny Yasser · Jimmy Lin · Amine Mhedhbi

Flock is a DuckDB extension that deeply integrates LLM and RAG capabilities into OLAP systems, so you can mix analytics with semantic analysis in SQL using map and reduce functions for generation, classification, embeddings, and more.

## Overview

Flock brings language models and retrieval into DuckDB so semantic analysis runs alongside ordinary SQL analytics.

<Columns cols={1}>
  <Card title="Administrative Views for Managing Prompts and Models" icon="database">
    Treat models and prompts as resources similar to tables. You can CREATE PROMPT(s) and MODEL(s) and iterate on them without making changes to SQL queries.
  </Card>
  <Card title="Scalar / Map Functions for Tuple and Passage Transformation" icon="filter">
    Map tuples or text passages to new text, boolean, or JSON values using LLMs to support various types of semantic analysis such as extraction and classification.
  </Card>
  <Card title="Aggregate / Reduce Functions to Reduce over Tuples and Passages" icon="layer-group">
    Use Aggregate / Reduce functions to reduce tuples or text passages using LLMs to support various types of semantic analysis such as reranking and summarization.
  </Card>
</Columns>

## Why Use Flock?

- Create reusable models and prompts once, then call them from simple SQL.
- Run LLM workloads next to your data without custom services or glue code.
- Keep results in tables you can join, filter, and visualize with regular DuckDB queries.

```sql
INSTALL flock FROM community;
LOAD flock;
```
