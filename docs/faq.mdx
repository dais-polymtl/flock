## General Questions

<AccordionGroup>
<Accordion title="What is Flock?">
Flock is an extension for DuckDB that brings semantic analysis directly into SQL queries. It integrates language models and retrieval-augmented generation through map and reduce functions, with support for text, image, and audio inputs.
</Accordion>

<Accordion title="Who can benefit from using Flock?">
Flock is designed for developers, data scientists, and businesses that need semantic analysis alongside traditional SQL. It is especially valuable for document ranking, content generation, multimodal analytics, and semantic search.
</Accordion>

</AccordionGroup>

## Using Flock

<AccordionGroup>
<Accordion title="How do I manage models in Flock?">
You can manage models using SQL commands like:
- `CREATE MODEL`
- `UPDATE MODEL`
- `DELETE MODEL`

Refer to [Model Management](/resource-management/models) for detailed examples, including `max_batch_size`, `is_async`, `rate_limit`, and `usage_limit`.
</Accordion>

<Accordion title="How do I set up secrets for providers?">
Check the [Secrets Management](/resource-management/api-keys) section for a step-by-step guide.
</Accordion>

<Accordion title="Can I create custom prompts for my tasks?">
Yes. Create and manage prompts with `CREATE PROMPT` and `UPDATE PROMPT`. See [Prompt Management](/resource-management/prompts) for details.
</Accordion>

<Accordion title="How do I use images or audio in queries?">
Use `context_columns` with `type: 'image'` for vision models or `type: 'audio'` with a `transcription_model` for audio. See [Image Support](/image-support) and [Audio Support](/audio-support).
</Accordion>

</AccordionGroup>

## Troubleshooting

<AccordionGroup>
<Accordion title="What should I do if a query fails?">
- Verify that the model and provider are correctly configured
- Ensure secret keys are valid and up to date
- Check input data for inconsistencies
- For multimodal queries, confirm `context_columns` use the correct `type` (`image` or `audio`) and required fields
</Accordion>

<Accordion title="How do I debug API-related errors?">
- Check provider usage limits and confirm the API key is active
- Use `GET SECRET` to confirm the correct key is in use
- Inspect provider error messages (OpenAI, Azure, Anthropic, etc.)
- Run `SELECT flock_get_metrics()` to review token usage and call counts
</Accordion>

<Accordion title="Why is the response time slow for some queries?">
Response times depend on:
- Provider API latency (OpenAI, Azure, Anthropic) or local hardware (Ollama)
- Query complexity and prompt size
- **`max_batch_size`**: larger batches mean fewer API calls but bigger payloads
- **`is_async`**: parallel batching (default) vs synchronous processing
- **`rate_limit`**: intentional throttling to stay within provider quotas
- Multimodal inputs (images, audio) increase payload size and processing time
- Context window limits: large datasets may trigger multiple API calls or automatic batch retries

See the [Performance and cost](/performance) guide for tuning recommendations.
</Accordion>

</AccordionGroup>
